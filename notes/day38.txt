프로젝트 방법론

    use-case 관계 : 포함과 확장
        1. 포함관계 (include)
            ex) 회원가입 시 주소검색 -> 주소 검색이 필수이므로 회원가입 중간에 주소검색 창으로 넘어감(?)
            회원가입 ----> 주소검색

        2. 확장관계 (extend) 
            ex) 회원가입 시 사진 등록 -> 특정 상황일 때 선택적으로 수행하는 작업
            회원가입 <---- 사진등록 


스택(stack)
    - LIFO (Last In First Out) 방식으로 데이터를 넣고 꺼낸다.
    - 데이터를 넣는 것을 'push'라고 하고, 데이터를 꺼내는 것을 'pop'이라고 한다.
    - 보통 입력한 역순으로 데이터를 꺼내야 하는 상황에서 이 자료구조를 사용한다.
        ex) JVM 스택 메모리 영역에서 메서드 호출을 관리할 때
            웹 브라우저에서 이전 페이지로 따라 올라갈 때
            breadcrumb 구현할 때
                - breadcrumb이란? => 빵부스러기로 표시한 길이라는 핸젤과 그레텔에서 따온 용어로,
                    사이트나 웹 앱에서 유저의 위치를 보여주는 부차적인 내비게이션 시스템을 뜻한다
